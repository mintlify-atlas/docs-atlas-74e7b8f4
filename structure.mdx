## Page navigation

The two pages link to each other through a single fixed navigation element: an anchor tag with the class `schedule`.

```html index.html
<!-- On the main page, navigates to the schedule -->
<a class="schedule" href="about.html">My Schedule</a>
```

```html about.html
<!-- On the schedule page, navigates back home -->
<a class="schedule" href="index.html">Home</a>
```

The `.schedule` class is defined in `index.css` and styles this button consistently on both pages. Because `about.html` imports both `about.css` and `index.css`, the nav button inherits the same appearance everywhere.

<Tip>
  Both pages share `index.css`. If you restyle the `.schedule` nav button, the change applies to both the home page and the schedule page simultaneously.
</Tip>

## CSS architecture

<CardGroup cols={2}>
  <Card title="css/index.css" icon="palette">
    The primary stylesheet at 1,007 lines. Covers the full `index.html` layout including the animated `headingText` list, hero image positioning, the `.busSect` work experience section, internship and project cards, the wave SVG divider, and the goodbye footer. Also defines the shared `.schedule` nav button used on both pages.
  </Card>
  <Card title="css/about.css" icon="table">
    A focused 32-line stylesheet for the schedule page only. Styles the `.about-main` container, the `<table>` element, and the `<th>`/`<td>` cells that display the weekly class grid.
  </Card>
</CardGroup>

`about.html` imports both stylesheets in order:

```html about.html
<link rel="stylesheet" href="css/about.css">
<link rel="stylesheet" href="css/index.css">
```

This means `index.css` rules can override `about.css` rules when selectors conflict. In practice the two files target distinct elements, so there is no conflict.
